## 0. What this is and why it is first

S0.1 is the **instrument**: the estimators and the synthetic ground truth that prove they work, before any real or simulated data is interpreted. It is the dependency root of the whole program — S0.2 (stability campaign), S0.3 (cascade definitions), and every later stage import it — and it is the only Stage-0 slice whose acceptance criterion lives **entirely on synthetic data**, with no dangling external dependency.

Phase 1 delivers exactly **Gate-0(a)** from the plan: *the scaling-relation check distinguishes a critical generator from a non-critical power-law look-alike*, plus recovery of planted `n` and `τ`. It maps onto the anti-drift contract's **Gate A** (criticality is earned) and **Gate D** (result-blind selection — every threshold here is set on synthetic data / compute budget, never on a result).

**The bounding rule (load-bearing).** Phase-1's stopping criterion is *"recovers planted `n` within CI at the planned event counts,"* **not** "maximally robust at `n=0.99`." The Hawkes likelihood genuinely flattens as `n→1`; "make the MLE robust at the boundary" is an open-ended optimization trap. The gate is deliberately built to be N-robust (§5) so it does **not** depend on winning that fight. The near-critical regime is *characterized and handed to S0.2*, not gated on here (§9).

## 1. Scope

**In scope (Phase 1):**

- The numeric thresholds (event-count floor, Gate-A.4 CI threshold, H1c `delta`, **Gate-A.2 informativeness ceiling**) → **S0.2**. Phase-1 uses the ceiling-free sign test and imports no S0.2 threshold.
- Kernel selection, mean-variance estimator, non-stationary baseline, long-memory kernel → **Phase 2** (sibling stub).
- Any real / Polymarket / on-chain data, any LLM or API, the data-feasibility spike → **S0.4** and later.

## 3. Package layout & unit interfaces

Two dataclasses in `types.py` are the **only** types modules exchange, so each unit is understood and tested in isolation:

```python
# critaudit/types.py  (illustrative)
@dataclass(frozen=True)
class EventStream:
    times: np.ndarray            # float64, strictly increasing
    marks: np.ndarray | None = None   # RESERVED for Phase 2; always None in Phase 1
    horizon: float | None = None      # observation window [0, horizon]

@dataclass(frozen=True)
class AvalancheSet:
    sizes: np.ndarray            # int/float, S per avalanche  (S >= 1)
    durations: np.ndarray        # int/float, T per avalanche  (T >= 1)
    censored: np.ndarray | None = None   # bool mask: avalanche hit the generator size cap
    # generator-native in S0.1; cascade-extracted in S0.3 — same type, different source.
    # `censored` flags generator avalanches truncated at the size cap; the scaling
    # estimators exclude them. None for cascade-extracted sets (S0.3).
```

`EventStream.marks` is reserved now (Phase 2 needs marked events) so the type does not change under Phase 2.

**Approved deviation from the plan's layout:** generators live **inside** the package (`critaudit.generators`), not a top-level `synthetic/`. Reason: `critaudit` detaches as a standalone toolkit; the generators are its validation surface and must ship with `pip install critaudit` so any downstream user (including later scale-free imaging/PET work — the reason it detaches at all) can validate their own pipeline against the same ground truth. Top-level `synthetic/` becomes the *run-and-writedown* location. (`DECISIONS.md` entry.)

## 4. Module specifications

### 4.1 `generators`

**G1 — `hawkes_sim.simulate(n, horizon, kernel="exp", backend="auto", rng)` → `EventStream`**
- Single exponential kernel `φ(t) = n·β·e^{−βt}`; branching ratio `n = ∫φ = α/β` (here parametrized directly by `n`, with `β` fixed/configurable). Immigrant (baseline) rate `μ` configurable.
- **Two backends:**
  - `"thinning"` — Ogata thinning. Used for `n ≤ 0.9`.
  - `"cluster"` — immigrant-cluster (Hawkes–Oakes) representation: exact for exponential kernels, degrades gracefully where thinning's acceptance rate collapses. Used for `n ≥ 0.9`. This is the bulk generator S0.2 leans on for the near-critical cells.
  - `"auto"` — thinning for `n ≤ 0.9`, cluster for `n ≥ 0.9`.
- **Generator-validity cross-check (required, §5):** at `n = 0.6` both backends are valid; the harness runs both and asserts agreement.

**G2 + CSN-killer — `branching.galton_watson(offspring_mean=m, n_avalanches, rng, size_cap=10**10)` → `AvalancheSet`**
- Poisson(`m`) offspring branching process; each tree is one avalanche with size `S` (total nodes) and duration `T` (generations). Simulated generation-by-generation (`Z_{g+1} ~ Poisson(m·Z_g)`), so per-avalanche cost is `O(T)`, not `O(S)`.
- **Size cap + censoring:** the critical (`m=1`) size distribution has infinite mean, so a `size_cap` bounds runaway avalanches; capped avalanches are flagged in `AvalancheSet.censored` and **excluded by the scaling estimators**. **The cap must clear the natural `s_max ~ N²`, or it truncates the upper tail and the CSN truncated-power-law LRT correctly rejects the critical sizes — failing Gate A.1.** `size_cap=10**10` keeps censoring ≈0 (and the tail intact) for `n_avalanches` up to ~1e5; a warning fires above 0.05% censored. ⚠️ **Corrected 2026-06-16:** the original `10**6` default was a build-time bug — it depleted the tail at only ~0.1% censored and made the critical G2 cell un-passable on A.1. *Censored-fraction is the wrong proxy; tail-integrity is the target.* See `DECISIONS.md` 2026-06-16.
- `m = 1` → **G2 critical**: exact mean-field targets `τ = 3/2`, `α = 2`, `1/σνz = 2`, with `(α−1)/(τ−1) = 2`.
- `m < 1` → **CSN-killer (Gate-A.1 tier)**: shares the `s^{−3/2}` body with an exponential cutoff at `s_c ~ 1/(1−m)²`; differs from G2 *only in the tail*. S0.1 runs it at `m ≈ 0.7` (clean, unambiguous CSN rejection). `m` is the exposed knob; S0.2 drives `m → 1` (barely-subcritical, hard-to-resolve cutoff) to calibrate the floor.

**Scaling-breaker — `lookalike.scaling_break(critical: AvalancheSet, delta, rng)` → `AvalancheSet` (Gate-A.2 tier)**
- Permutes the `(S, T)` pairing of a critical `AvalancheSet`. `delta` controls the degree; `delta → 1` is the **full shuffle**.
- At full shuffle (S0.1's setting): marginals untouched (A.1 still passes, `τ≈3/2`, `α≈2`), `⟨S⟩(T)` goes flat → measured `1/σνz ≈ 0` → `Δ = (α−1)/(τ−1) − 1/σνz ≈ 2`. A **huge, N-robust** violation by design.
- `delta` is the exposed knob; S0.2 drives small-`delta` (a *small* real scaling violation) to calibrate the A.2 informativeness ceiling and false-positive rate against genuine criticality.

> Both look-alikes ship out of S0.1 as **parametrized** generators on G2's GW backbone. S0.1 uses the clean extremes; S0.2 reuses the same generators at near-threshold settings through the same scaling API — no rebuild.

### 4.2 `hawkes`

**`mle.fit(events: EventStream, kernel="exp", rng=None)` → `HawkesFit(n, params, loglik, ci)`**
- Exponential-kernel log-likelihood via **Ogata's O(N) recursion** (the recursive sum, no O(N²) double loop). `scipy.optimize` for the MLE. `n` reported as the kernel integral.
- **Stationarity-unconstrained:** the optimizer does **not** constrain `n < 1`. The pre-registration treats `n ≥ 1` as a reportable outcome ("fitted Hawkes may be non-stationary"), so `n̂` may land above 1. This is *why* the CI must be asymmetric.
- **CI = profile-likelihood (primary):** invert the LRT (χ² threshold); the upper limit is free to exceed 1. Wald/Fisher (symmetric) is **rejected** — wrong shape at the boundary. The CI bounds are found by **`scipy.optimize.brentq` root-finding** on `2·(profile_nll(n) − nll_min) − χ²₁,₀.₉₅`, *not* a fine grid scan — verified necessary because the sequential Ogata recursion makes a 200-point profile scan (≈40k recursion passes) untenable, while root-finding needs ≈30 profile evaluations per fit.
- The failure mode being audited at `n = 0.99` is **variance inflation from a flat likelihood**, *not* boundary non-regularity (there is no Chernoff-type χ² breakdown because `n < 1` is not imposed).

**`residuals.rescaled_residuals(events, fit)` → KS diagnostic** — time-rescaling (Ogata) residuals; transformed inter-event times should be unit-rate exponential. Phase-1 uses this as a goodness-of-fit readout; Phase-2's `kernel_select` is built on the same machinery.

### 4.3 `powerlaw`

**`csn.fit_powerlaw(x)` → `PowerLawFit(exponent, xmin, p_boot, lrt)`; `.passes -> bool`** — thin wrapper over the `powerlaw` package: MLE exponent, KS-selected `x_min`, parametric-bootstrap goodness-of-fit `p`, LRT vs lognormal / exponential / truncated power-law. **Gate-A.1 PASS** = bootstrap `p ≥ p*` **and** no LRT significantly favours an alternative, with **`p* = 0.10` fixed by the CSN (2009) convention** — result-blind (a methodological standard, not an inspected result), set now, *not* deferred to S0.2. For the harness's two A.1 cases the verdict is N-robust and invariant to `p* ∈ {0.05, 0.10}`: the CSN-killer is rejected because the **LRT vs the truncated power law significantly favours the truncated alternative** (its cutoff at `s_c ~ 1/(1−m)²` is real and detected), which needs no `p*` at all; G2 passes because no LRT favours an alternative and its bootstrap `p` sits comfortably above `p*`.

### 4.4 `scaling`

**`crackling.exponents(av: AvalancheSet)` → `(tau, alpha, inv_snz)`**, and **`crackling.delta(av)` → `float`** = `(α−1)/(τ−1) − 1/σνz`.
- `τ` = CSN exponent of `sizes`; `α` = CSN exponent of `durations` (both via `powerlaw.Fit`, KS-selected `x_min`).
- `1/σνz` = the **curvature-corrected `⟨S|T⟩` slope** (`log⟨S|T⟩ = a + γ logT + b/T`, report `γ`; censored avalanches excluded). This is a *reasonable* estimator, **not a load-bearing one** — Gate A.2 is a **coarse corroborative check** (below), so the estimator need only be sensible, not optimal.

> **Gate A.2 is coarse and corroborative, by measurement (`DECISIONS.md` 2026-06-15).** A genuinely critical generator gives `Δ → 0` with N (+0.17 at 2e4 → +0.01 at 4e5; the earlier "bias floor" was single-seed noise, withdrawn). But A.2's *resolution* `Δ_min(N) = 2√2·SD_seed(Δ)` is **coarse (~0.24 at 1e5, ~0.12 at 4e5) and intrinsic** — dominated by the CSN marginal-exponent (`τ, α`) seed-variance that `(α−1)/(τ−1)` amplifies, **not** the `1/σνz` estimator. (A joint `(τ,α)` estimator is the one un-exhausted lever; measured to be implausible to close the ~order-of-magnitude gap to fine resolution — see `DECISIONS.md`.) So A.2 **cannot** be a fine quantitative gate at achievable counts, and is not made one: it corroborates the `n`-based criticality claim (the headline) by confirming the scaling relation is *not grossly broken*. This is **more** "earned, not asserted," not less — asserting a precision the instrument cannot deliver was the over-reach.

**`crackling.joint_bootstrap(av: AvalancheSet, B, rng)` → `DeltaResult(samples, ci, halfwidth)`** — the S0.2 interface (§7). Resamples **avalanches** (not exponents) and re-estimates all three exponents per replicate so their correlations are preserved in `Δ`.
> **Hard requirement on the signature:** each replicate re-runs the **full CSN pipeline including KS `x_min` selection**. Fixing `x_min` from the full-data fit and resampling only above it conditions away `x_min`-selection uncertainty and yields a too-narrow `halfwidth` — which would bias S0.2's A.2 ceiling tight and make A.2 over-declare "consistent" on real data. This is the precise failure the anti-drift contract exists to prevent.

**`gate_a2.corroborate(delta: DeltaResult, resolution: float, k=2.0)` → str** — **coarse corroboration**, not a fine pass/fail. `"not-grossly-violated"` if `|Δ| ≤ k·resolution`; `"grossly-violated"` if `|Δ|` exceeds it. `resolution` is the reported `Δ_min(N)`. There is **no** reference-null, no informativeness ceiling, no fine trichotomy — those were the over-reach the discrimination probe retired (`DECISIONS.md` 2026-06-15). A.2 confirms the scaling relation is not grossly broken for a system already identified as near-critical via `n`; it does **not** adjudicate near-critical gradations.

The parameter exists now so Phase-1 imports no S0.2 threshold and S0.2 needs no API change.

## 5. The Gate-0(a) validation harness (`validation/gate0a.py`)

A **2×2 truth table**, asserted **deterministically (seeded)**. The harness is built to be **N-robust and calibration-independent** — its verdicts do not ride on a CI threshold that is only settled in S0.2.

| Generator (extreme setting) | Gate A.1 (CSN) | Gate A.2 (sign test) |
|---|---|---|
| G2 critical (`m=1`) | **pass** | **not-inconsistent** (CI contains 0) |
| CSN-killer (`m=0.7`) | **reject** | — |
| Scaling-breaker (full shuffle) | **pass** | **inconsistent** (CI excludes 0; `1/σνz≈0`, `Δ≈2`) |

- The `Δ≈2` violation is *far* outside any CI — that hugeness is the **feature** that makes the gate N-robust.
- The scaling-breaker's A.1-pass is **definitionally identical** to G2's (same size multiset after shuffling, same CSN computation) — that cell cannot flake.
- The CSN-killer's A.1-reject rests on the **LRT vs the truncated power law** (the `s_c ~ 1/(1−m)²` cutoff is real and detected), not on a finely-tuned `p*` — N-robust by construction (§4.3).
- **A.2 critical cell is BLOCKED on the §4.4 estimator.** The scaling-breaker "CI excludes 0" cell (`Δ≈2`) is robust under any reasonable `1/σνz` estimator. The G2 "CI contains 0" cell is **not** achievable with the binned-slope estimator (it false-negatives criticality, §4.4) and awaits the shape-collapse replacement; until then this cell is reported **blocked/inconclusive**, not a pass or fail. The harness logs the realized avalanche count and censored fraction in the writedown.
- **Generator-validity cross-check (required):** at `n = 0.6`, run G1 with **both** backends and assert agreement on (i) recovered `n` and (ii) summary event statistics — event count, mean inter-event time, avalanche-size histogram — within a concrete tolerance: the two backend means agree within `2×` the Monte-Carlo SE across the `R` realizations, and the size histograms are not separated by a two-sample test at a fixed level. Rationale: `auto` never exercises both paths on a shared target, yet the unvalidated `cluster` path feeds every high-`n` cell S0.2 builds on (including the ceiling calibration). A silent bug there would corrupt the entire near-critical regime.

**Writedown artifact.** The harness writes a calibration certificate to `results/s0.1_gate0a/<date>_writedown.txt` (the `preflight/` convention from the owner's global checklist): measured `τ, α, 1/σνz, Δ` and CIs per generator, every pass/fail cell, the backend cross-check deltas, and the `n`/`τ` recovery summary (§6). The act of writing the numbers down is part of the gate, not a side effect.

## 6. Acceptance thresholds (the recovery criterion)

Stated as **empirical coverage** against known truth (the right framing for an instrument), not single-realization checks.

**Grid (result-blind — set by compute budget only):**
- `n`-recovery (Hawkes): `n ∈ {0.3, 0.6, 0.9, 0.99}` × `N_events ∈ {1e3, 1e4}`, `R = 100` realizations/cell.
- Monotonicity sanity: `N_events = 1e5` at `n ∈ {0.3, 0.6}` **only** (not at 0.99 — expensive and redundant with S0.2's floor sweep).
- `τ`-recovery (GW): `n_avalanches ∈ {1e3, 1e4}`, `R = 100`.
- Realizations are seed-spawned (`SeedSequence.spawn`) and run via **`multiprocessing`** (embarrassingly parallel; the per-fit Ogata recursion stays sequential).

**`R = 100`** gives coverage SE ≈ 0.03 — enough to separate 0.90 from 0.80, which is all a gate needs.

**Pass criteria (Gate-0(a)):**
1. The 2×2 truth table (§5) holds, deterministically.
2. **`n`-recovery:** 95%-profile-CI coverage of planted `n` ≥ **0.90** at every cell with `n ∈ {0.3, 0.6, 0.9}` and `N ≥ 1e4`.
3. **`τ`-recovery:** CSN `τ̂` 95% CI contains 1.5 for critical GW at `n_avalanches ≥ 1e4`.
4. The `n=0.6` thinning↔cluster backend cross-check passes.
5. Coverage is monotone non-decreasing in `N` at `n ∈ {0.3, 0.6}` (sanity).
6. The writedown artifact is emitted.

**The near-critical cell (`n = 0.99`) is characterized, not gated.** The harness reports its coverage, CI **width**, and the profile↔bootstrap agreement (below) — but its acceptance *threshold* is S0.2's deliverable (the near-critical floor), not a Phase-1 gate. Honest behavior here means: when the instrument cannot resolve `n`, the profile CI is **wide** (low precision), not falsely narrow. Wide-but-covering at `0.99` is correct; coverage falling below 0.90 there is a *finding* written down and handed to S0.2, and is exactly what the audit below interrogates.

**Profile↔bootstrap audit at `n = 0.99` (method transfer, not calibration).** Coverage-vs-ground-truth already certifies whether profile-likelihood is honest at the boundary, so the parametric bootstrap is **not** a calibration check. Its job: the bootstrap is the CI we will use on **real data, where there is no ground truth** (Stage 1). The audit confirms profile and bootstrap **agree on shared datasets at `n = 0.99`**, so the approach carries forward. The writedown records "profile↔bootstrap agreement at the boundary," not "validate profile coverage."

## 10. Deferred to S0.2 (named here so nothing silently imports a missing threshold)

The Gate-A.2 **informativeness ceiling**, the event-count **floor**, the Gate-A.4 **CI threshold**, and the H1c **`delta`** are **all S0.2 deliverables**, set by result-blind synthetic stability simulations that *consume* S0.1's `joint_bootstrap` and parametrized look-alikes. Phase-1 deliberately runs only the regimes where verdicts are unambiguous, so it imports none of them. (The CSN `p* = 0.10` is *not* on this list — it is fixed now by the CSN convention; see §4.3.)
