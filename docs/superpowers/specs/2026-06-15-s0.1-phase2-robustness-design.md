## Why this is a separate file

Phase 2 is what you need to fit **real** data honestly — *not* what you need to validate the instrument. It is a **Stage-1 prerequisite**, not a Gate-0(a) prerequisite. A single document with a fully-detailed Phase 1 beside a sketched Phase 2 invites an agent to scaffold Phase-2 stubs before Phase-1 is validated, which is exactly the trap the firewall closes. The file boundary is an enforceable scope gate.

Its validation is still synthetic, so it remains within S0.1's remit — but as a strictly later phase.

## What it will cover (to be specified later)

- **`hawkes/kernel_select.py`** — the random-time-change (Ogata residual → KS) kernel-selection *rule*, frozen as a procedure. Selects exponential vs power-law/long-memory kernel on a **held-out** segment; the selected kernel's MLE `n` is primary.
- **`hawkes/` long-memory kernel** — power-law / sum-of-exponentials self-excitation, extending the Phase-1 Ogata recursion through the reserved extension point (no change to `mle.fit`'s contract or to `EventStream`).
- **`hawkes/meanvar.py`** — Hardiman–Bouchaud mean-variance branching-ratio approximator (robustness estimator; divergence from MLE is a finding, never averaged).
- **`hawkes/baseline.py`** — EM piecewise-constant + logspline **non-stationary** baseline (and a semimartingale baseline if feasible). Estimated on **held-out** structure.

## Synthetic validation targets (the Phase-2 analog of Gate-0(a))

- **Kernel selection picks the true kernel.** Plant exponential vs long-memory ground truth; the rule recovers which, on held-out data, above chance by a pre-set margin.
- **Recover `n` under a planted non-stationary baseline.** Generate events with a known time-varying `μ(t)`; the EM/logspline baseline + kernel recovers planted `n` within CI, where a constant-baseline fit would inflate it.
- **Identifiability sensitivity (the Filimonov–Sornette confound).** Quantify how much a long-memory kernel and a non-stationary baseline trade off; report the sensitivity, and confirm the **Gate-A.3** rule fires (reject "critical" when `n≈1` only under the constant baseline).
